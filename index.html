Each level adds more axioms/constraints. Fields are the richest structure.

Group G = {S, ·}
Mnemonic

CAIN Axioms

  • Closure — a,b ∈ G ⟹ a·b ∈ G
  • Associativity — (a·b)·c = a·(b·c)
  • Identity — ∃ e such that a·e = e·a = a
  • Nverses — ∃ a⁻¹ such that a·a⁻¹ = e
Abelian Group

+ Commutativity

  • All CAIN axioms hold
  • Additionally: a·b = b·a
  • The integers ℤ under addition form an abelian group
Cyclic Group

Generator Concept

  • Every element is a power of some fixed element a
  • b = aᵏ for some k, for every b in group
  • a is called the generator
  • a³ = a·a·a (repeated application)
Formal Group Axioms
A1Closure — If a,b ∈ G then a·b ∈ G
A2Associativity — a·(b·c) = (a·b)·c for all a,b,c ∈ G
A3Identity — ∃ e ∈ G such that a·e = e·a = a for all a
A4Inverse — For each a ∈ G ∃ a' such that a·a' = a'·a = e
A5Commutative — a·b = b·a (only for Abelian groups)
Ring R = {S, +, ×}
Ring Axioms

R is Abelian Group for Addition + Multiplication rules

  • A1–A5 for addition (abelian group)
  • M1: Closure under ×
  • M2: Associativity of ×
  • M3: Distributive — a(b+c) = ab+ac
Commutative Ring

+ M4

  • M4: ab = ba for all a,b ∈ R
Integral Domain
  • M5: Multiplicative identity — a·1 = 1·a = a
  • M6: No zero divisors — ab=0 ⟹ a=0 or b=0
Field F = {S, +, ×}
Field = Integral Domain + Multiplicative Inverse for every non-zero element.
M7: For each a ∈ F (a≠0), ∃ a⁻¹ such that a·a⁻¹ = a⁻¹·a = 1
Field Properties

Both operations form abelian groups

  • Abelian group under addition
  • Abelian group under multiplication (ignoring 0)
  • Satisfies A1–A5, M1–M7
Key Insight

Hierarchy

Group ⟶ Ring ⟶ Field More axioms ⟹ Richer structure Fields most useful in cryptography
02

Galois Fields

Key Fact: Finite fields play a key role in cryptography. The number of elements in a finite field must be a prime power pⁿ. These are called Galois Fields GF(pⁿ).
GF(p)

Prime Fields

  • Set of integers {0, 1, …, p−1}
  • Arithmetic operations modulo prime p
  • Forms a finite field — multiplicative inverses exist
  • Can do +, −, ×, ÷ without leaving the field
GF(7) = {0, 1, 2, 3, 4, 5, 6} All arithmetic mod 7
GF(2ⁿ)

Binary Extension Fields

  • Polynomials with binary coefficients
  • Degree less than n
  • Reduction modulo an irreducible polynomial of degree n (for multiplication)
  • Can always find an inverse using Extended Euclid

03

Polynomial Arithmetic

General Form

Polynomial

f(x) = aₙxⁿ + aₙ₋₁xⁿ⁻¹ + … + a₁x + a₀ = Σ aᵢxⁱ

x is the indeterminate — we don't evaluate at a specific value

Three Levels

Alternatives

  • Ordinary polynomial arithmetic
  • Poly arithmetic with coefficients mod p
  • Poly arithmetic with coefs mod p AND polynomials mod m(x)
Ordinary Polynomial Arithmetic
let f(x) = x³ + x² + 2 and g(x) = x² – x + 1 f(x) + g(x) = x³ + 2x² – x + 3 (add corresponding coefficients) f(x) – g(x) = x³ + x + 1 f(x) × g(x) = x⁵ + 3x² – 2x + 2 (multiply all terms)
Polynomial Arithmetic with Coefficients mod 2
Binary Coefficients

Mod 2 Poly Arithmetic

  • All coefficients are 0 or 1
  • Addition = XOR of coefficients
  • Subtraction = Addition (mod 2)
f(x) = x³ + x², g(x) = x² + x + 1 f(x) + g(x) = x³ + x + 1 f(x) × g(x) = x⁵ + x²
Division

Polynomial Division

  • f(x) = q(x)·g(x) + r(x)
  • r(x) = f(x) mod g(x)
  • If r(x)=0, then g(x) divides f(x)
Irreducible polynomial — g(x) has no divisors other than itself and 1 (analogous to a prime number).
Arithmetic modulo an irreducible polynomial forms a field.
GCD for Polynomials — Euclidean Algorithm
Euclid(a(x), b(x)): if (b(x) = 0) then return a(x); else return Euclid(b(x), a(x) mod b(x)); c(x) = GCD(a(x), b(x)) if c(x) is the poly of greatest degree that divides both a(x) and b(x)
04

Public-Key Cryptography

🔐 Private-Key (Symmetric)

  • One shared key for encrypt & decrypt
  • Both parties must know the key
  • If key is disclosed → compromised
  • Fast computation

🔑 Public-Key (Asymmetric)

  • Two keys: public & private
  • Public key: anyone can encrypt / verify
  • Private key: owner decrypts / signs
  • Proposed by Diffie & Hellman, 1976
The Six Ingredients
Ingredient 1

Plaintext

The readable message or data fed as input to the algorithm.

Ingredient 2

Encryption Algorithm

Performs transformations on the plaintext using the key.

Ingredient 3

Public & Private Keys

A pair: if one encrypts, the other decrypts. Exact transformations depend on which key is provided.

Ingredient 4

Ciphertext

Scrambled output. Depends on both the plaintext and the key. Two different keys → two different ciphertexts.

Ingredient 5

Decryption Algorithm

Accepts ciphertext and matching key to recover the original plaintext.

Applications — Three Categories
Category 1

Encryption / Decryption

Provides secrecy. Encrypt with public key, decrypt with private key.

Category 2

Digital Signatures

Provides authentication. Sign with private key, verify with public key.

Category 3

Key Exchange

Exchange session keys. Some algorithms support all uses; others are specific.

Why Public-Key? Two key problems it addresses:
1. Key Distribution — how to have secure comms in general without prior shared key
2. Digital Signatures — how to verify a message comes intact from the claimed sender
05

RSA Algorithm

RSA — Rivest, Shamir & Adleman, MIT, 1977. Best known & widely used public-key scheme. Based on exponentiation in a finite (Galois) field over integers modulo a prime. Security due to the difficulty of factoring large numbers.
Key Setup — Step by Step
01

Select two large primes

Choose large primes p and q at random

02

Compute modulus n

n = p × q

03

Compute Euler's totient φ(n)

φ(n) = (p−1)(q−1)

04

Select encryption key e

Choose e where 1 < e < φ(n) and gcd(e, φ(n)) = 1

05

Compute decryption key d

Solve e·d ≡ 1 mod φ(n), 0 ≤ d ≤ n

06

Publish / Keep secret

Public key: PU = {e, n} | Private key: PR = {d, n}

Encryption & Decryption
ENCRYPT: C = Mᵉ mod n (sender uses public key {e, n}) DECRYPT: M = Cᵈ mod n (owner uses private key {d, n}) Note: Message M must be smaller than modulus n
Why RSA Works — Euler's Theorem
Euler's Theorem: aφ(n) mod n = 1 (where gcd(a,n)=1) Proof of Correctness: Cᵈ = (Mᵉ)ᵈ = Mᵉ·ᵈ = M1+k·φ(n) = M¹ · (Mφ(n))ᵏ = M¹ · (1)ᵏ = M mod n ✓
Worked Example
Key Setup

p=17, q=11

n = 17 × 11 = 187 φ(n) = 16 × 10 = 160 e = 7 (gcd(7,160)=1) d = 23 (7×23 mod 160 = 1) PU = {7, 187} PR = {23, 187}
Encrypt/Decrypt M=88

Encrypt then Decrypt

C = 88⁷ mod 187 = 11 Calculation: 88¹ mod 187 = 88 88² mod 187 = 77 88⁴ mod 187 = 132 88⁷ mod 187 = (88×77×132) mod 187 = 11 M = 11²³ mod 187 = 88 ✓
Exam Practice Numericals
Problem 1

C=10, e=5, n=35. Find M

n=35=5×7 → p=5, q=7 φ(n) = 4×6 = 24 e=5 → d: 5d ≡ 1 mod 24 → d=5 M = 10⁵ mod 35 = 10
Problem 2

p=3, q=11, e=7, M=5

n = 3×11 = 33 φ(n) = 2×10 = 20 d: 7d ≡ 1 mod 20 → d=3 C = 5⁷ mod 33 = 78125 mod 33 = 14
Problem 3

p=5, q=11, e=3, M=9

n = 5×11 = 55 φ(n) = 4×10 = 40 d: 3d ≡ 1 mod 40 → d=27 C = 9³ mod 55 = 729 mod 55 = 14
Square and Multiply Algorithm (Fast Exponentiation)
Algorithm

Only O(log₂ n) multiplications

c = 0; f = 1 for i = k downto 0: c = 2 × c f = (f × f) mod n if bᵢ == 1: c = c + 1 f = (f × a) mod n return f
Example

Using binary exponent

  • 7⁵ = 7⁴·7¹ = 3·7 = 10 mod 11
  • 3¹²⁹ = 3¹²⁸·3¹ = 5·3 = 4 mod 11
Common choices for e: 65537 (= 2¹⁶+1), or 3 or 17. Using e=3 can be attacked via Chinese Remainder Theorem.
RSA Security Attacks
Attack 1

Brute Force

Try all possible private keys. Infeasible for 1024–2048 bit keys.

Attack 2

Mathematical (Factoring)

  • Factor n = p·q → compute φ(n) → compute d
  • Or determine φ(n) directly
  • Or find d directly
  • Currently: assume 1024–2048 bit RSA is secure
Attack 3

Timing Attacks

  • Paul Kocher, mid-1990s
  • Exploit timing variations in operations
  • Infer operand size from time taken
  • Countermeasures: constant-time exponentiation, random delays, blind values
Attack 4

Chosen Ciphertext (CCA)

  • Attacker chooses ciphertexts and gets decrypted plaintext back
  • Exploits multiplicative properties of RSA
  • Counter: random padding of plaintext
  • Or use OAEP (Optimal Asymmetric Encryption Padding)
06

Diffie-Hellman Key Exchange

First public-key type scheme proposed. Diffie & Hellman, 1976 (Williamson at UK CESG secretly proposed it in 1970).
"Astounding concept: two parties can carry on a public conversation and still end up with a secret that only they know!"
Key Idea

Public Secret Sharing

  • Cannot exchange arbitrary messages
  • Establishes a common secret key
  • Known only to the two participants
  • Used as session key in symmetric encryption
Security Basis

Discrete Logarithm Problem

  • Easy: compute y = aˣ mod q
  • Hard: given y, compute x (discrete log)
  • Similar difficulty to factoring
Protocol
SETUP: All users agree on global parameters: q = large prime a = primitive root mod q ALICE: Chooses secret xₐ < q Computes public key: yₐ = aˣᴬ mod q BOB: Chooses secret x_B < q Computes public key: y_B = aˣᴮ mod q SHARED SESSION KEY: K_AB = aˣᴬˣᴮ mod q Alice computes: K_AB = y_B^xₐ mod q Bob computes: K_AB = yₐ^x_B mod q (Same result!)
Worked Example
Prime q = 353, Primitive root a = 3 Alice: xₐ = 97 → yₐ = 3⁹⁷ mod 353 = 40 Bob: x_B = 233 → y_B = 3²³³ mod 353 = 248 Alice: K_AB = 248⁹⁷ mod 353 = 160 Bob: K_AB = 40²³³ mod 353 = 160 ✓
Man-in-the-Middle (MITM) Attack
Also called "Bucket Brigade" Attack. Both random and static D-H keys are vulnerable.
1

Darth Prepares

Creates two separate private/public key pairs (one for each end)

2

Alice Sends Public Key

Alice transmits her public key yₐ to Bob

3

Darth Intercepts

Darth intercepts yₐ, sends his first public key to Bob instead. Darth calculates a shared key with Alice.

4

Bob Responds

Bob receives Darth's key and calculates a shared key — with Darth, not Alice

5–7

Mirror on Bob's Side

Bob sends his key; Darth intercepts and sends second key to Alice. Alice calculates shared key with Darth.

Result: Darth has a private, unauthenticated channel with each of Alice and Bob. Can intercept, decrypt, re-encrypt, and forward all messages.

Countermeasures: Public Key Infrastructure (PKI), PGP web of trust, or scheduling/timing-based prevention.
07

ElGamal Cryptography

ElGamal is a public-key cryptosystem related to Diffie-Hellman. Uses exponentiation in a finite field. Security based on difficulty of computing discrete logarithms.
Key Generation
Alice's Key: Chooses secret xₐ: 1 < xₐ < q−1 Computes public key: yₐ = aˣᴬ mod q
Encryption (Bob encrypts for Alice)
Message M in range [0, q−1] (longer → send as blocks) Choose random integer k, 1 ≤ k ≤ q−1 One-time key: K = yₐᵏ mod q Ciphertext pair (C₁, C₂): C₁ = aᵏ mod q // like D-H public key C₂ = K·M mod q // encrypted message
Decryption (Alice recovers M)
Recover K: K = C₁^xₐ mod q Recover M: M = C₂ · K⁻¹ mod q ⚠️ A unique K must be used each time — otherwise insecure!
Worked Example — GF(19), a=10
Alice: xₐ = 5, yₐ = 10⁵ mod 19 = 3 Bob sends M = 17, chooses k = 6: K = 3⁶ mod 19 = 7 C₁ = 10⁶ mod 19 = 11 C₂ = 7·17 mod 19 = 5 Sends: (11, 5) Alice decrypts: K = 11⁵ mod 19 = 7 K⁻¹ = 7⁻¹ mod 19 = 11 (7×11 = 77 ≡ 1 mod 19 ✓) M = 5·11 mod 19 = 17 ✓

Elliptic Curve Cryptography

ECC — an alternative to RSA/D-H using elliptic curves. Offers same security with much smaller key sizes. Newer but not as well analysed.
Real Elliptic Curve

Cubic Form

y² = x³ + ax + b x, y, a, b ∈ ℝ Also define zero point O

Addition P+Q: reflect the intersection R with the x-axis. Geometric interpretation.

Finite Elliptic Curves

Two Families

  • Prime curves Ep(a,b) — defined over Zp, integers mod prime. Best in software.
  • Binary curves E₂ₘ(a,b) — defined over GF(2ⁿ), binary polynomial coefficients. Best in hardware.
The Hard Problem — Elliptic Curve Discrete Log
EASY: Given k, P → compute Q = kP (k times point addition) HARD: Given Q, P → find k (Elliptic Curve Logarithm Problem) This is the security foundation of ECC. Example: E₂₃(9, 17)
ECC Diffie-Hellman
Select curve Eq(a,b) and base point G = (x₁, y₁) with large order n such that nG = O Alice: nₐ < n (private), Pₐ = nₐG (public) Bob: n_B < n (private), P_B = n_BG (public) Shared key: K = nₐ·P_B = n_B·Pₐ = nₐ·n_B·G ✓
ECC Encryption / Decryption
Encode message M as point Pₘ on the curve. Each user has private nₐ and public Pₐ = nₐG. Encrypt: Cₘ = {kG, Pₘ + kP_B} (k = random) Decrypt: Pₘ + kP_B − n_B(kG) = Pₘ + k(n_BG) − n_B(kG) = Pₘ ✓

PRNG Based on Asymmetric Ciphers
PRNG from RSA

Micali-Schnorr PRNG

  • Uses RSA-based exponentiation
  • Specified in ANSI X9.82 and ISO 18031
  • Much slower than symmetric PRNG
  • Use only to generate short bit sequences (e.g., keys)
PRNG from ECC

Dual Elliptic Curve PRNG

for i = 1 to k: sᵢ = x(sᵢ₋₁ · P) rᵢ = lsb₂₄₀(x(sᵢ · Q)) return r₁, …, rₖ

NIST SP 800-9. Some controversy on security/efficiency. Use only if just have ECC.

Attacks & Standards

Attacks on Public Key Cryptography
RSA Specific

Factoring Attack

  • Progress in integer factoring directly threatens RSA
  • Best algorithm: Lenstra Sieving — 200 decimal digits (663 bits) as of 2005
  • Safe: 1024–2048 bit RSA
Any Public Key

Side Channel Attacks

  • Timing attacks (Paul Kocher)
  • Power analysis attacks
  • Exploit physical implementation, not the math
Quantum Threat

Quantum Computing

  • Breaks RSA with key-length computation
  • Breaks ECC with 2× key-length computation
  • Shor's algorithm can factor n in polynomial time
Any — Limited Plaintext

Cube-Root Attack (RSA e=3)

  • If message is "small" and m³ < n
  • Then just take cube root of c to get m!
  • Fix: require e > 3, or pad m to make it "big"
  • PKCS#1 prevents this
Limited Plaintext Attack (any scheme): If only a few plaintexts are possible, encrypt all of them and look for a match. Countermeasure: require a random "confounder" in plaintext.
